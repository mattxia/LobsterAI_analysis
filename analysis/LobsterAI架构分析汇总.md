# LobsterAI 架构分析汇总

## 一、分析概述
本次分析针对 LobsterAI 桌面 AI 助手应用的整体技术架构和核心子系统进行了全面梳理，涵盖了从前端UI到后端核心引擎的完整技术栈，重点分析了系统分层架构、核心组件职责、通信协议以及沙箱安全系统的设计与实现。

## 二、整体技术架构总结

### 1. 分层架构
LobsterAI 采用经典的 Electron 三层架构设计：
```
┌─────────────────────────────────────────┐
│  Renderer 层 (React + Redux + TypeScript) │
│  - UI组件、状态管理、业务服务层          │
└─────────────────────┬───────────────────┘
                      │ IPC 通信
┌─────────────────────▼───────────────────┐
│  Main 层 (Node.js + Electron)           │
│  - 进程管理、IPC路由、核心业务逻辑      │
│  - 存储引擎、Agent引擎、技能系统、IM网关│
└─────────────────────┬───────────────────┘
                      │ 外部调用
┌─────────────────────▼───────────────────┐
│  外部依赖层                             │
│  - LLM API、OpenClaw引擎、系统调用      │
│  - 第三方服务、自定义技能               │
└─────────────────────────────────────────┘
```

### 2. 核心组件矩阵
| 层级 | 核心组件 | 主要职责 |
|------|----------|----------|
| **渲染层** | CoworkView、CoworkService、Redux Store | 用户交互、会话管理、状态同步、UI渲染 |
| **主进程 - 核心层** | SqliteStore、CoworkStore、CoworkEngineRouter | 数据持久化、会话存储、多引擎调度 |
| **主进程 - 引擎层** | CoworkRunner、ClaudeRuntimeAdapter、OpenClawRuntimeAdapter | AI会话执行、引擎适配、工具调用管理 |
| **主进程 - 子系统** | SkillManager、IMGatewayManager、CronJobService、MemorySystem | 技能管理、多IM接入、定时任务、用户记忆 |
| **沙箱层** | OpenClawEngineManager、SandboxGateway、SecurityScanner | 安全隔离执行、权限控制、安全审计 |

### 3. 架构设计亮点
1. **引擎无关设计**：通过统一的引擎抽象层，可无缝切换内置Claude引擎和OpenClaw沙箱引擎，上层业务无感知
2. **严格安全隔离**：渲染进程与主进程完全隔离，contextIsolation和sandbox完全开启，仅通过预定义IPC通道通信
3. **事件驱动架构**：核心组件基于EventEmitter实现，松耦合且易于扩展
4. **模块化设计**：各业务领域独立封装，边界清晰，便于维护和功能扩展
5. **统一流式响应**：所有AI响应采用统一的流式事件框架，保证UI体验一致性
6. **可扩展技能系统**：基于目录结构的技能自动加载机制，新增技能无需修改核心代码

## 三、沙箱系统专项总结

### 1. 沙箱架构
沙箱系统基于OpenClaw引擎实现，采用网关-代理架构：
```
┌─────────────────┐     IPC      ┌──────────────────────────┐
│  渲染层权限弹窗  │ ◄───────────► │  OpenClawRuntimeAdapter  │
└─────────────────┘              └─────────────┬────────────┘
                                                │ WebSocket
┌──────────────────────────┐                   ▼
│  SkillSecurityScanner    │        ┌──────────────────────────┐
│  （脚本审计、调用审计）   │        │  OpenClaw 沙箱网关       │
└──────────────────────────┘        └─────────────┬────────────┘
                                                │ 隔离执行
                                      ┌─────────▼─────────┐
                                      │  沙箱执行环境      │
                                      │  (文件系统隔离、   │
                                      │   网络隔离、       │
                                      │   资源限制)        │
                                      └───────────────────┘
```

### 2. 核心能力
| 能力项 | 说明 |
|--------|------|
| **执行模式** | 支持三种模式：`local`(本地直接执行)、`auto`(部分隔离)、`sandbox`(完全隔离) |
| **通信协议** | 主进程与沙箱通过WebSocket双向通信，支持流式响应和事件推送 |
| **权限控制** | 细粒度工具调用审批，支持按命令类型、访问路径、风险等级设置权限 |
| **安全审计** | 三层安全检查：技能脚本审计、工具调用审计、输出内容审计 |
| **隔离级别** | 支持全隔离、部分隔离、无隔离三种级别，可根据场景动态切换 |

### 3. 关键流程
1. **会话启动流程**：用户选择沙箱模式 → 引擎路由到OpenClaw适配器 → 启动沙箱 runtime → 同步配置 → 创建会话 → 执行用户请求
2. **权限审批流程**：Agent请求工具调用 → 沙箱发送权限请求 → UI弹窗提示用户 → 用户批准/拒绝 → 沙箱执行/终止操作
3. **安全隔离流程**：工具请求 → 安全扫描 → 沙箱模式判断 → 对应隔离级别执行 → 结果安全审计 → 脱敏返回

### 4. 当前状态
当前版本（2026.3.x）沙箱功能已完成框架设计，但默认关闭（所有命令本地执行），后续版本将逐步开放完整的隔离能力。

## 四、技术栈总结
| 领域 | 技术选型 |
|------|----------|
| **前端框架** | React 18 + TypeScript + Tailwind CSS |
| **状态管理** | Redux Toolkit |
| **桌面框架** | Electron 28+ |
| **数据存储** | SQL.js (SQLite) |
| **AI引擎** | Claude Agent SDK / OpenClaw |
| **构建工具** | Vite + electron-builder |
| **测试框架** | Vitest |
| **多语言支持** | 自研i18n服务（中/英） |
| **安全能力** | 内置技能安全扫描器、沙箱隔离、权限审批 |

## 五、后续演进建议
1. **沙箱能力开放**：逐步启用完全沙箱隔离模式，支持自定义沙箱策略
2. **多引擎扩展**：增加更多AI引擎支持（如Gemini、DeepSeek等）
3. **插件生态**：完善MCP（模型上下文协议）生态，支持更多第三方工具接入
4. **性能优化**：优化大会话加载速度、降低内存占用
5. **协同能力**：支持多用户会话协同、团队共享记忆库
